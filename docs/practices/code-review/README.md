# Code Review Playbook (Sub-Directory)

Use this folder during PR reviews to answer three questions quickly:

1. **Where do tests live, and what do they cover?** (directory + contract mapping)
2. **When should I run which tests?** (based on what you changed)
3. **Where do I look when things fail?** (failures vs violations vs summary)
